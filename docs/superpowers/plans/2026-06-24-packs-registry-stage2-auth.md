# PackHub Registry — Stage 2: Authentication (Better Auth) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Google + GitHub OAuth and email magic-link login (email as primary identity) backed by Cloudflare D1, with login state visible on every page.

**Architecture:** Better Auth 1.5 with native D1 (`betterAuth({ database: env.DB })`). Because the D1 binding only exists per-request on Workers, the auth instance is built by a **per-request factory** `createAuth(env)` — never at module scope. A catch-all SSR route `/api/auth/[...all]` delegates to `auth.handler`. The auth tables are generated by the Better Auth CLI against a throwaway local SQLite (the CLI can't reach D1) and applied to D1 as a wrangler migration. Marketing pages stay prerendered, so the NavBar reads login state **client-side** via the Better Auth browser client. Magic-link email sends through Resend in prod and logs the link to the console in local dev.

**Tech Stack:** Better Auth 1.5 (`better-auth`), Cloudflare D1, Astro 5 SSR (`@astrojs/cloudflare` v12), Resend (email), `better-sqlite3` (devDep, CLI schema gen only).

## Global Constraints

- Package manager **pnpm**; Node `22.22.0`. Conventional Commits, header ≤ 100 chars.
- Import alias `~/*` → `src/*`. TypeScript Astro **strict**; ESLint `--max-warnings=0`.
- i18n strings live in BOTH `src/i18n/en/translation.json` and `src/i18n/zh/translation.json`, structurally identical (shallow top-level merge).
- Cloudflare bindings: `DB` (D1), `CACHE` (KV) — already declared in `wrangler.toml` (Stage 1).
- Secrets only in `.dev.vars` (gitignored) / Cloudflare env — never committed. Keys: `BETTER_AUTH_SECRET`, `BETTER_AUTH_URL`, `GOOGLE_CLIENT_ID/SECRET`, `GITHUB_OAUTH_CLIENT_ID/SECRET`, `RESEND_API_KEY` (see `.dev.vars.example`).
- **Email = primary identity**; account linking unifies Google/GitHub/magic-link on verified email.
- SSR routes set `export const prerender = false`; all existing pages stay `prerender = true`.
- Every task ends green on the relevant subset of: `pnpm typecheck`, `pnpm lint`, `pnpm format`, `pnpm spell`, `pnpm test`, `pnpm build`.

```ts
export async function sendMagicLinkEmail(
  env: Env,
  { email, url }: { email: string; url: string }
): Promise<void> {
  if (!env.RESEND_API_KEY) {
    // Dev fallback: no email provider configured.
    console.log(`[magic-link] ${email} -> ${url}`);
    return;
  }
  const res = await fetch('https://api.resend.com/emails', {
    method: 'POST',
    headers: {
      Authorization: `Bearer ${env.RESEND_API_KEY}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      from: env.RESEND_FROM ?? 'NevoFlux <login@nevoflux.app>',
      to: email,
      subject: 'Sign in to NevoFlux',
      html: `<p>Click to sign in: <a href="${url}">${url}</a></p>`,
    }),
  });
  if (!res.ok) throw new Error(`Resend failed: ${res.status}`);
}
```

```ts
import type { BetterAuthOptions } from 'better-auth';
import { magicLink } from 'better-auth/plugins';
import { sendMagicLinkEmail } from '~/lib/email';

/** Shared options. `database` is the D1 binding (prod) or a sqlite Database (CLI). */
export function buildAuthOptions(
  env: Env,
  database: unknown
): BetterAuthOptions {
  return {
    database: database as BetterAuthOptions['database'],
    secret: env.BETTER_AUTH_SECRET,
    baseURL: env.BETTER_AUTH_URL,
    account: {
      accountLinking: { enabled: true, trustedProviders: ['google', 'github'] },
    },
    socialProviders: {
      google: {
        clientId: env.GOOGLE_CLIENT_ID,
        clientSecret: env.GOOGLE_CLIENT_SECRET,
      },
      github: {
        clientId: env.GITHUB_OAUTH_CLIENT_ID,
        clientSecret: env.GITHUB_OAUTH_CLIENT_SECRET,
      },
    },
    plugins: [
      magicLink({
        sendMagicLink: async ({ email, url }) => {
          await sendMagicLinkEmail(env, { email, url });
        },
      }),
    ],
  };
}
```
